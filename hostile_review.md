# QKeyboard — 적대적 프로젝트 진행 상황 리뷰

이 문서는 QKeyboard 프로젝트의 현재 진행 상황과 문제점을 극도로 비판적이고 냉정하게 분석한 결과물입니다.

## 1. 코어 모델 안정성 및 QA 프로세스 부재
- **화면 렌더링조차 안 되는 코드를 방치:**
  `KeyboardController::rowsForPage()`에서 `result.append(rowList)` 호출로 인해 2차원 행 구조가 1차원 키 배열로 완전히 깨져서 반환되던 치명적인 코어 버그가 존재했습니다.
  - 이로 인해 QWidget 뷰(`KeyboardWidget`)와 QML 뷰(`KeyboardPanel.qml`)는 사실상 키보드를 전혀 화면에 그리지 못하거나 크래시가 발생하는 수준이었으나, 이슈 #11 유닛 테스트 확장을 진행하기 전까지 누구도 발견하지 못했습니다. 화면 구동조차 검증하지 않고 이전 이슈들을 마구잡이로 완료 처리했던 방만한 개발 태도를 방증합니다.

## 2. 에이전트 간 소통 부재 및 중복 작업 (리소스 낭비)
- **한글 IME 오토마타 중복 구현:**
  이슈 #9(한글 입력 조합기) 작업을 로컬에서 진행하는 과정에서, 다른 에이전트(Claude)가 이미 PR #28을 통해 `HangulComposer`의 전체 구현을 완료하고 `main` 브랜치에 병합(Merge)해 두었다는 사실을 전혀 감지하지 못했습니다.
  - 이로 인해 동일한 한글 오토마타 동작 로직을 구현하는 데 수십 단계에 걸쳐 중복 코딩을 수행하였고, 중복 PR(#43)을 생성했다가 반려/삭제하는 불필요한 작업 리소스 낭비를 유발했습니다. 작업 전 원격 PR 및 브랜치 상태 모니터링 절차가 완전히 무너져 있었음을 보여줍니다.

## 3. 취약한 크로스 플랫폼 지원 및 검증
- **플랫폼 검증의 지연:**
  기존의 CI 워크플로우는 오로지 Linux(`ubuntu-latest`) 환경에서만 테스트 및 빌드가 수행되고 있었습니다. 크로스 플랫폼 지원 라이브러리(Qt)를 표방하면서 정작 다른 OS에서의 정상 작동 여부를 사전에 파악할 수 있는 시스템이 없었습니다.
  - 그 결과 PR #31에서 OS 매트릭스를 추가하자마자 macOS 환경에서 레거시 그래픽 라이브러리(`AGL`) 참조 문제로 빌드가 즉시 깨졌습니다. 개발 전반에 걸쳐 다양한 플랫폼 호환성에 대한 세심한 대비가 전혀 부재한 상태로 방치되어 있었습니다.

## 4. 키보드 핵심 기능의 결손 (깡통 프레임워크 상태)
현재 13개 이상의 기초적인 키보드 기능(Shift 고정, 백스페이스 반복 입력, 특수 키 피드백, 각국 언어 IME 레이아웃 등)이 전부 구현 예정이거나 홀딩된 상태입니다.
- **백스페이스 길게 누르기 반복 삭제(Long-press Repeat, #32) 전무:** 사용자가 글자를 지우기 위해 버튼을 직접 연타해야 하는 구시대적 UI 상태입니다.
- **Shift One-shot/Locked 모드(#33) 부재:** 모바일 가상 키보드에서 제공하는 표준 더블 클릭 Shift 고정 기능이 없습니다.
- **입력 피드백 피처(#38) 전무:** 키를 눌렀을 때의 효과음, 햅틱 진동, 또는 입력 반응을 알 수 있는 팝업 피드백이 아예 설계되어 있지 않습니다.
- **다국어 레이아웃/IME 스펙 공백:** 한/영을 제외하고 일본어(#29), 중국어(#30) 등 동아시아 3개국 언어의 주요 IME 구현 계획이 여전히 공백으로 남겨져 있습니다.

## 결론
현재 프로젝트는 **"키보드 버튼 레이아웃 형상을 대략 띄우고, 텍스트가 흘러가는 것을 간신히 흉내 내는 정적 버튼 그리드"**에 불과합니다. 기본적인 다국어 입력 장치 규격을 완전히 충족하고 안정적인 UI 성능을 내기까지는 아직도 무수한 버그 수정과 기반 피처의 전면적인 재설계가 요구되는 매우 심각한 지연 상태입니다.
